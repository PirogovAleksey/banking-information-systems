Завдання 1.2: Підготовка середовища для аналізу

Необхідні інструменти та налаштування:

Для статичного аналізу:
  • APKTool - декомпіляція Android додатків 
    apktool d app.apk -o output_folder
  • jadx - декомпілятор Java/Kotlin коду 
    jadx -d output app.apk
  • MobSF (Mobile Security Framework) - автоматизований аналіз 
    docker run -it -p 8000:8000 opensecurity/mobile-security-framework-mobsf
  • Hopper/IDA Pro - для iOS додатків (опціонально)
Для динамічного аналізу:
  • Android Studio з емулятором - тестове середовище
  • Burp Suite / OWASP ZAP - перехоплення трафіку 
    # Налаштування проксі в емуляторі
adb shell settings put global http_proxy 192.168.1.100:8080
  • Frida - runtime instrumentation (для досвідчених) 
    frida -U -f com.bank.app -l script.js --no-pause
  • adb (Android Debug Bridge) - взаємодія з пристроєм 
    # Основні команди
adb devices
adb install app.apk
adb logcat | grep "BankApp"
adb shell pm list packages

Налаштування безпечного тестового середовища:

  1. Використовуйте ізольоване середовище:
    • Окрема віртуальна машина або емулятор
    • Тестовий Google/Apple акаунт
    • Тестовий банківський акаунт (якщо можливо)
  2. Налаштуйте мережеву ізоляцію:
    • Використовуйте VPN або окрему мережу
    • Налаштуйте проксі для перехоплення трафіку
    • Встановіть CA сертифікат для HTTPS interception
  3. Документуйте всі дії:
    • Робіть скріншоти кожного кроку
    • Зберігайте логи та network traces
    • Фіксуйте версії додатку та ОС

Результат виконання:

  • Обраний додаток з обґрунтуванням вибору
  • Налаштоване тестове середовище
  • Встановлені та налаштовані інструменти аналізу
  • Створений тестовий акаунт (якщо можливо)

Частина 2: Статичний аналіз додатку

Завдання 2.1: Аналіз структури та компонентів додатку

Що потрібно зробити:

Провести декомпіляцію додатку та проаналізувати його структуру, компоненти, permissions та потенційні вразливості в коді.

Покрокова інструкція для Android:

  1. Отримання APK файлу: 
    # Спосіб 1: З пристрою
adb shell pm list packages | grep bank
adb shell pm path com.bank.app
adb pull /data/app/com.bank.app/base.apk

# Спосіб 2: З онлайн сервісів (APKPure, APKMirror)
  2. Декомпіляція з APKTool: 
    apktool d banking_app.apk -o decompiled_app/
cd decompiled_app/
ls -la  # Огляд структури
  3. Аналіз AndroidManifest.xml:
    Що шукати в маніфесті:
    • Permissions: Які дозволи запитує додаток? 
      <uses-permission android:name="android.permission.CAMERA"/>
<uses-permission android:name="android.permission.READ_PHONE_STATE"/>
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
    • Exported components: Чи є відкриті активності/сервіси? 
      <activity android:name=".MainActivity" android:exported="true">
    • Backup settings: Чи дозволено резервне копіювання? 
      android:allowBackup="false"
android:fullBackupContent="false"
    • Debug mode: Чи вимкнено debug режим? 
      android:debuggable="false"
  4. Аналіз коду з jadx: 
    jadx -d jadx_output banking_app.apk
# Відкрийте jadx-gui для візуального аналізу

Чек-лист для статичного аналізу:

Частина 3: Динамічний аналіз додатку

Завдання 3.1: Аналіз мережевого трафіку

Що потрібно зробити:

Налаштувати перехоплення HTTPS трафіку між мобільним додатком та сервером для аналізу API запитів, автентифікації та передачі даних.

🔴 КРИТИЧНО ВАЖЛИВО: Перехоплення трафіку проводиться ТІЛЬКИ для власного тестового акаунту в навчальних цілях. Перехоплення чужого трафіку є кримінальним злочином!

Налаштування Burp Suite для Android:

  1. Налаштування Burp Proxy:
    • Proxy → Options → Add: Bind to address: All interfaces, Port: 8080
    • Отримайте IP адресу вашого комп'ютера: ipconfig (Windows) або ifconfig (Linux/Mac)
  2. Встановлення CA сертифікату на Android: 
    # Експорт сертифікату з Burp
Proxy → Options → Import/Export CA Certificate → Export → DER format

# Конвертація для Android
openssl x509 -inform DER -in burp.der -out burp.pem
openssl x509 -inform PEM -subject_hash_old -in burp.pem | head -1
mv burp.pem 9a5ba575.0  # Використайте отриманий хеш

# Встановлення на емулятор (root required)
adb push 9a5ba575.0 /sdcard/
adb shell
su
mount -o rw,remount /system
cp /sdcard/9a5ba575.0 /system/etc/security/cacerts/
chmod 644 /system/etc/security/cacerts/9a5ba575.0
  3. Налаштування проксі на пристрої: 
    # Для емулятора
Settings → Network → Advanced → Manual proxy
Host: [IP вашого комп'ютера]
Port: 8080

# Або через adb
adb shell settings put global http_proxy 192.168.1.100:8080

Обхід Certificate Pinning (навчальні цілі):

Методи обходу pinning:
  • Для Android 7.0+ (Network Security Config): 
    # Декомпілюйте APK
# Додайте в res/xml/network_security_config.xml:
<network-security-config>
    <base-config>
        <trust-anchors>
            <certificates src="user" />
            <certificates src="system" />
        </trust-anchors>
    </base-config>
</network-security-config>

# Перекомпілюйте та підпишіть APK
  • Використання Frida (потребує root): 
    # JavaScript скрипт для Frida
Java.perform(function() {
    var CertificatePinner = Java.use('okhttp3.CertificatePinner');
    CertificatePinner.check.overload('java.lang.String', 'java.util.List').implementation = function() {
        console.log('Certificate Pinning bypassed');
    };
});
  • Objection Framework: 
    objection -g com.bank.app explore
android sslpinning disable

Завдання 5.3: Compliance та стандарти

Відповідність регуляторним вимогам:

Обов'язкові стандарти для мобільного банкінгу в Україні:
  • НБУ Постанова №95 - Кібербезпека в банківській системі
    • Обов'язкова двофакторна автентифікація
    • Шифрування чутливих даних
    • Логування всіх операцій
    • Incident response процедури
  • PCI DSS - для карткових операцій
    • Не зберігати CVV2/CVC2
    • Маскування PAN (показувати тільки останні 4 цифри)
    • Tokenization для карткових даних
  • PSD2 Strong Customer Authentication (SCA)
    • Два з трьох факторів: knowledge, possession, inherence
    • Dynamic linking для транзакцій
    • Exemptions для low-risk операцій
  • GDPR - захист персональних даних
    • Privacy by Design
    • Data minimization
    • Right to erasure

❓ Часті запитання

1. Чи можна аналізувати закордонний банківський додаток?

Так, але рекомендується обрати український для кращого розуміння локального контексту та вимог НБУ.

2. Що робити якщо додаток має сильний захист від аналізу?

Документуйте всі спроби обходу, опишіть захисні механізми - це теж важлива частина аналізу.

3. Чи обов'язково використовувати root/jailbreak?

Ні, багато аналізу можна провести без root. Але з root доступом аналіз буде глибшим.

4. Як довго зберігати дані аналізу?

Видаліть всі чутливі дані одразу після здачі роботи. Не зберігайте токени, паролі, персональні дані.

5. Що робити якщо знайшли критичну вразливість?

НЕ експлуатуйте її! Опишіть в звіті та повідомте викладача. За responsible disclosure можна повідомити банк.
